### ğŸ” Term Retrospective

**Focus:** Core IT literacy and technical environment awareness.

This term established non-programming foundations required for software work: operating systems, networking, and data fundamentals. While not development-heavy, these courses built the mental model for how software interacts with real infrastructure.

#### Key Skills Developed 
- Understanding system-level constraints (OS, hardware, networking)
- Reading technical documentation instead of relying on tutorials
- Foundational SQL and relational database theory
- Translating technical communication to non-technical audiences.

#### What I Would Do Differently
At the time, I treated these courses as "supporting courses." In hindsight, I would spend more time mapping these concepts directly to application development (e.g., how networking affects APIs, or how OS scheduling impacts performance).

I also would have maybe chosen some other courses to take first to start with more momentum. These courses (especially Data Management - Foundations, Hardware and Operating Systems Essentials, and Networking Security) were very information heavy. While all of this was important knowledge to have, it felt a little overwhelming at times trying to navigate actually retaining all the knowledge instead of memorizing for the test.

#### How This Term Shows Up in My Work Now
This foundation influences how I design backend systems â€” especially thinking about:
- Environment configuration
- Deployment context
- Data persistence strategy
